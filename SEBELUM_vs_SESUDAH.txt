# SEBELUM vs SESUDAH - Fitur Penjualan Multi-Item

## 🔴 SEBELUM Implementasi

### Form Create
```
❌ Hanya bisa pilih 1 barang saja
❌ Tidak ada tombol "Tambah Item"
❌ Jika mau jual 2 barang, harus buat transaksi 2x
❌ User experience: Ribet untuk penjualan banyak barang

FORM LAYOUT:
┌─────────────────────────┐
│ Pilih Barang (dropdown) │
│ Jumlah (input)          │
│ [Bayar]                 │
└─────────────────────────┘
```

### Index View
```
❌ Jika jual 3 barang → tampil 3 rows (per item)
❌ Sulit memahami transaksi mana saja yang related
❌ Tidak ada breakdown detail per item
❌ User harus search/scroll untuk tracking

TABLE:
| No | Barang    | Jumlah | Total  |
|----|-----------|--------|--------|
| 1  | Barang A  | 2 pcs  | 20000  |
| 2  | Barang B  | 3 pcs  | 30000  |
| 3  | Barang C  | 1 pcs  | 10000  |
(Terlihat seperti 3 transaksi terpisah)
```

### Database
```
❌ no_transaksi UNIQUE constraint
   → Tidak bisa simpan 2 items dengan no_transaksi sama
   → Harus gunakan nomor transaksi berbeda

TABLE transaksis:
id | no_transaksi | barang_id | jumlah | harga | ...
1  | TRX-0001     | 1         | 2      | 10000 | ✓
2  | TRX-0002     | 2         | 3      | 10000 | ✓ (ERROR kalau TRX-0001)
```

## 🛡️ Data Integrity Improvements

### SEBELUM
```
❌ Jika cancel item B dari transaksi:
   - Item A masih "selesai"
   - Item B bisa "batal"
   - Item C masih "selesai"
   → Inconsistent state

❌ Jika ada duplicate items, tidak ter-handle baik
❌ Stok validation tidak comprehensive
```

### SESUDAH
```
✅ Cancel transaksi:
   - Semua items (A, B, C) status "batal"
   - Semua stok ter-restore sesuai jumlah
   - Atomic operation (all or nothing)

✅ Duplicate items di-detect & warn user
✅ Stock validation: combined qty check
✅ Stock deduction: combine + decrement once
```

## 💾 Database Migration Impact

### PERUBAHAN TABEL
```sql
-- SEBELUM (tidak support multi-item)
ALTER TABLE transaksis ADD CONSTRAINT uq_no_transaksi UNIQUE(no_transaksi);

-- ERROR saat insert:
INSERT INTO transaksis (no_transaksi, barang_id, ...) VALUES ('TRX-001', 1, ...);
INSERT INTO transaksis (no_transaksi, barang_id, ...) VALUES ('TRX-001', 2, ...);
-- ❌ Duplicate entry 'TRX-001' for key 'uq_no_transaksi'

-- SESUDAH (support multi-item)
ALTER TABLE transaksis DROP CONSTRAINT uq_no_transaksi;
ALTER TABLE transaksis ADD INDEX idx_no_transaksi(no_transaksi);

-- OK saat insert:
INSERT INTO transaksis (no_transaksi, barang_id, ...) VALUES ('TRX-001', 1, ...);
INSERT INTO transaksis (no_transaksi, barang_id, ...) VALUES ('TRX-001', 2, ...);
-- ✅ Success
```

### DATA EXISTING
```
✅ Backward compatible
✅ No data loss
✅ Existing transaksis dengan unique no_transaksi tetap work
✅ Bisa add multiple items ke existing no_transaksi? Tidak (sudah unique)
   → Hanya untuk transaksi baru
```
