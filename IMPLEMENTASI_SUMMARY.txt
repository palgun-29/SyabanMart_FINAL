## 🔍 Validasi & Logic

### Stock Validation (Multi-Item)
```
SCENARIO: Barang A dipilih 2x (qty 3 + qty 5, stok 10)

STEP 1: Form submission dengan 2 items
STEP 2: Combine quantities: Barang A → 8 pcs total
STEP 3: Validate: 8 <= 10 ✓ OK
STEP 4: Save 2 items dengan no_transaksi sama
STEP 5: Decrement stok once: A stok jadi 2

JIKA STOK KURANG:
Barang A qty 3 + qty 8 = 11 pcs, stok = 10
→ Error: "Barang 'Barang A': stok tidak cukup (stok: 10 pcs, dibutuhkan: 11 pcs)"
→ Form kembali, tidak ada yang disimpan
```

### Cancel/Destroy Logic
```
TRANSAKSI: TRX-20260613-0001
- Item 1: Barang A, qty 5
- Item 2: Barang B, qty 3

SAAT CANCEL:
STEP 1: Find all items dengan no_transaksi = TRX-20260613-0001
STEP 2: Update status ke "batal" untuk semua items
STEP 3: Increment stok:
  - Barang A: +5
  - Barang B: +3
STEP 4: Trigger stock notifications jika perlu
STEP 5: Redirect dengan success message
```

## 📞 Troubleshooting

### Issue: "Unique constraint violated"
**Solusi:** Migration belum dijalankan
```bash
php artisan migrate
```

### Issue: Duplicate items tidak ter-combine
**Solusi:** JavaScript belum load / browser cache
```
Clear browser cache Ctrl+Shift+Del → reload page
```

### Issue: Stok tidak berkurang
**Solusi:** Check database connection, review logs
```bash
php artisan tinker
> App\Models\Barang::find(1)->stok
```

### Issue: Form data hilang saat error
**Solusi:** Ini normal, old input seharusnya ter-restore
```
Check: old('barang_id'), old('jumlah') di form
```
